# Guide de contribution

Merci de votre intérêt pour contribuer à ce guide Claude Code !

## Comment contribuer

### Rapporter des erreurs
- Vérifiez que l'erreur n'a pas déjà été signalée
- Créez une issue avec un titre descriptif
- Incluez des détails sur l'erreur trouvée

### Proposer des améliorations
- Créez une issue pour discuter de votre proposition
- Attendez les retours avant de commencer le travail

### Soumettre des modifications

1. Fork le projet
2. Créez une branche pour votre feature (`git checkout -b feature/AmazingFeature`)
3. Committez vos changements (`git commit -m 'Add some AmazingFeature'`)
4. Push vers la branche (`git push origin feature/AmazingFeature`)
5. Ouvrez une Pull Request

## Standards de documentation

### Style d'écriture
- Utilisez un français clair et accessible
- Préférez les phrases courtes
- Incluez des exemples concrets

### Formatage Markdown
- Utilisez les titres de manière hiérarchique
- Incluez des blocs de code avec syntaxe highlighting
- Ajoutez des liens vers d'autres sections pertinentes

### Structure
- Chaque section doit avoir un README.md
- Les exemples doivent être testés
- Incluez des références aux fichiers sources quand pertinent

## Code de conduite

- Soyez respectueux et constructif
- Acceptez les critiques avec bienveillance
- Concentrez-vous sur ce qui est meilleur pour la communauté

## Questions

Si vous avez des questions, n'hésitez pas à créer une issue.
